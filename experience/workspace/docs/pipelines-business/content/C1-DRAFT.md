## Engine Orchestration

| Step | Agent | Action |
|---|---|---|
| C1.1 | R-CONTENT | Pick voice variant per channel (3 voices per `16-brand §2`). |
| C1.2 | R-CONTENT | Apply template (article / LinkedIn / Twitter VN / newsletter / case study). |
| C1.3 | R-CONTENT | Substance check — ≥ 1 specific tactic/number/decision. |
| C1.4 | R-CONTENT | Citation pass — every stat URL-linked. |
| C1.5 | R-CONTENT | Lint — no AI tells, no hype words, ≤ 30% template fill.

## Outputs
```
projects/_business/marketing/drafts/
├── articles/YYYY-MM-DD-{slug}.md
├── social/{platform}-{slug}.md
├── newsletter/YYYY-MM-DD-{issue}.md
└── case-studies/{anonymized}.md
```

## Definition of Done
- ✅ Voice variant matches channel
- ✅ ≥ 1 client-specific or project-specific anchor
- ✅ All stats cited (URL or "no data" honesty)
- ✅ CTA present
- ✅ Lint pass: no "delve/unleash/tapestry/in today's"

## Failure Modes
- **Generic advice**: enforce concrete-example requirement
- **Hallucinated stats**: no URL → omit number
- **Voice drift**: variant mismatch → block
