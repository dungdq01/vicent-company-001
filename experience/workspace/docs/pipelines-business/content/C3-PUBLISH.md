# C3 — Publish

> Distribute per channel. Track engagement.

**Owner**: CEO · **Agent**: R-MKT (orchestrate) + R-CONTENT (cross-post copy)

---

## Engine Orchestration

| Step | Agent | Action |
|---|---|---|
| C3.1 | R-MKT | Pick channel(s) per topic + voice. |
| C3.2 | R-CONTENT | Generate channel-specific cross-post copy. |
| C3.3 | Human | Manual publish (Substack, LinkedIn, Twitter, blog). Tools auto-publish only with explicit allow-list. |
| C3.4 | R-MKT | Track engagement 24h, 7d, 30d. |
| C3.5 | R-MKT | Cohort tag: link clicks → newsletter subs → leads (via R-SDR feedback). |

## Channels per Type

| Type | Primary | Secondary |
|---|---|---|
| Article | Substack newsletter | Blog · LinkedIn long-form |
| Case study | Blog | LinkedIn · Twitter thread |
| Social — technical | Twitter VN | LinkedIn (dev community voice) |
| Social — biz | LinkedIn | Twitter EN (occasional) |
| Talk | YouTube + Substack repost | Conference page |

## Definition of Done
- ✅ Published on planned date (calendar adherence ≥ 80%)
- ✅ Cross-post copy distinct (not identical reshare)
- ✅ Tracking pixel / UTM in place
- ✅ Engagement check at 24h flagged if anomaly

## Failure Modes
- **Auto-publish bug**: hard allow-list; nothing posts without human ✓.
- **Identical cross-post**: voice drift; require channel-specific lead-in.
- **Tracking missing**: post counted as zero; engineering action.
