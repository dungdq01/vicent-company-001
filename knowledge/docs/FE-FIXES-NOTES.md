## Fix 1: Report Display — Center Cards, Not Sidebar

**Current:** Clicking sub-node → sidebar slides in from right → markdown rendered in narrow column.  
**Problem:** Feels cramped, not "smart" or cinematic. Content is squished into sidebar.

**Required:**
- When user clicks a sub-node in Planetary View → show report as **center overlay cards**
- Cards should be **centered on screen** (max-width ~800px), with dark glass background + green glow border
- Each major section (h2) of the report = **one card** in a **horizontal scroll carousel**
- User can **swipe/scroll left-right** between cards (like a presentation)
- Each card has **shadow + glassmorphism** effect: `bg-[#081210]/90 backdrop-blur-md border border-[#00FF88]/10`
- **Hide/seek accordion** within each card for sub-sections (h3, h4) — click to expand, click again to collapse
- Close button (X) or click outside to dismiss → back to Planetary View
- Transition: card appears with **scale-up + fade-in** animation (200ms)

**Files to modify:**
- `components/node/SubNodeDetail.tsx` — complete rewrite of layout
- `components/layout/Sidebar.tsx` — may need alternative "center overlay" mode
- `page.tsx` — handle overlay state instead of sidebar for planetary sub-nodes
- `globals.css` — add card animations, glassmorphism classes

**Reference:** Like the card grid in the reference image (Clandestine security dashboard) — each metric/feature is its own card with shadow and glow.

## Fix 3: Markdown Rendering Needs Dedicated Module

**Current:** `SubNodeDetail.tsx` has a 250-line inline `MarkdownRenderer` function. It handles basic cases but:
- Raw markdown syntax still visible in some edge cases (nested lists, complex tables, links)
- No image rendering support
- No link rendering (URLs not clickable)
- No horizontal rule styling consistency

**Required:**
- Extract `MarkdownRenderer` into its own file: `components/ui/MarkdownRenderer.tsx`
- Use `react-markdown` + `rehype-raw` + `remark-gfm` (already in project deps per MASTER-ARCHITECTURE)
- Custom components for the neon theme:
  - `h1-h4` → green/cyan glow text
  - `code` → dark card with green text
  - `pre` → dark card with syntax highlighting (consider `rehype-highlight`)
  - `table` → styled with green borders and hover rows (current table styling is good, port it)
  - `a` → cyan color with underline, opens in new tab
  - `blockquote` → green left border
  - `hr` → green gradient line
  - `img` → rounded with border glow
- Test with ALL existing report files in `docs/reports/B01/`

**Files to create:**
- `components/ui/MarkdownRenderer.tsx` — shared markdown component

**Files to modify:**
