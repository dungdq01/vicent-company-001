# High-Performer Hooks

> **STUDIO-INTERNAL ONLY**. Per `_shared/standards/boundaries.md` §1 + Path C §C7 (memory + high-performer save).
>
> Library of viral / high-engagement content hooks. Mined from Path C content pieces that exceeded engagement targets. Strategic agents (R-CONTENT, R-MKT) can read this for tone calibration; client-delivery agents cannot.

**Owner**: P3 / R-MKT mines, CEO sign on promotion to here.

---

## Format

```
## {hook-id: HH-{YYYYMMDD}-{slug}} · {channel: TikTok | YouTube | Substack | LinkedIn | X | Threads}

**Voice**: voice_a | voice_b | voice_c
**Hook (verbatim)**: "..."
**Format**: 1-3 sec hook | 5-second cold open | thread-1 tweet | substack subject line | etc.
**Performance**: views/CTR/engagement vs baseline (number + unit)
**Why it worked**: 1-sentence hypothesis (curiosity gap, contrarian, social proof, etc.)
**Reusability**: pattern OR one-time
**Pattern template**: if reusable, abstract to template
```

## Promotion criteria

- Performance ≥ 2× baseline of similar content (per `business-strategy/03 §KPIs`)
- Replicable pattern (not just one-time luck)
- Voice-aligned (per `voice-registry.yaml`)
- CEO sign before adding here

---

## Anti-Patterns (do NOT save)

- ❌ Banned word list hooks (delve, leverage, synergy, game-changer — per `40-docs-rules.md`)
- ❌ Fake hype / unrealistic promises (R-MAS-11)
- ❌ Hooks that broke voice contract (off-brand spike)
- ❌ Click-bait without substance follow-through
